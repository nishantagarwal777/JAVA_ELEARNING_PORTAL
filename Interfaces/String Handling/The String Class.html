The String Class
  • In Java, each character in a string is a 16-bit Unicode character.
  • In Java, strings are objects.
  • String objects are stored in a special memory area known as string constant pool inside the Heap memory.
  • The java.lang.String class has thirteen constructors that allow you to provide the initial value of the string using different sources.
  • The most direct way to create a string by creating a string literal:
      String hello=”Hello”;

    Each time you create a string literal, the JVM checks the string constant pool first. If the string does not exist in the pool, a new string object instantiates and then is placed in the pool.If the string already exists in the pool, a reference to the pooled instance returns.
      String hello=”Hello”;
  String helloAgain=”Hello”;//no new object will be created
  • An instance of a String can also be created with the new keyword, as follows:
      String hello = new String();//Creates 1 object and 1 reference variable
  hello=”Hello”;//Creates 1 object
    The first line of code creates a new object of java.lang.String class, and assigns it to the reference variable hello. The second line of code assigns the value “Hello” to hello.
  • A more efficient shortcut for creating a string is by providing the value of the string in the constructor. For example:
      String hello = new String(”Hello”); //Creates 2 object and 1 reference variable
  • Another way of creating a string is by creating an array of characters and then providing the name of the character array in the constructor. For example:
      char[] helloArray = { 'h', 'e', 'l', 'l', 'o', '.' };
  String hello = new String(helloArray);
  • String objects are immutable in nature.
  • The java.lang.String class is final in nature.
  • The String objects can be used with the += and + operators for concatenation.
