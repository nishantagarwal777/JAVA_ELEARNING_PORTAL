The StringBuffer Class
  • The java.lang.StringBuffer class should be used when you have to make a lot of modifications to strings of characters because StringBuffer objects are mutable in nature.
  • The StringBuffer class is same as String except it is mutable i.e. its value can be changed.
  • A common use for StringBuffers is file I/O when large, ever-changing streams of input are being handled by the program. In these cases, large blocks of characters are handled as units, and StringBuffer objects are the ideal way to handle a block of data, pass it on, and then reuse the same memory to handle the next block of data.
  • StringBuffer class is thread-safe i.e. multiple threads cannot access it simultaneously .So it is safe and will result in synchronization.
  • The java.lang.StringBuffer class is final in nature.
  • Some commonly used constructors of StringBuffer class are:
    1. StringBuffer(): creates an empty string buffer with the initial capacity of 16.
    2. StringBuffer(String str): creates a string buffer with the specified string.
    3. StringBuffer(int capacity): creates an empty string buffer with the specified capacity as length.
  • Some commonly used methods of StringBuffer class are:

    public synchronized StringBuffer append(String s): It is used to append the argument passed with the invoking string. The append() method is overloaded like append(char), append(boolean), append(int), append(float), append(double) etc. 
      //Example of append() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuffer sb = new StringBuffer("set ");
		sb.append("point");
		System.out.println(sb); // output is "set point" 

		StringBuffer sb2 = new StringBuffer("pi = ");
		sb2.append(3.14159f);
		System.out.println(sb2); // output is "pi = 3.14159"
	}
  }

      //Example of insert() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuffer sb = new StringBuffer("01234567");

    public int capacity(): It is used to return the current capacity of the invoking StringBuffer object. The default capacity of the buffer is 16. If the length of the object becomes more than the current capacity of the object due to modification then the capacity increases to (currentcapacity*2) +2. 
      //Example of capacity() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuffer sb= new StringBuffer();
		System.out.println(sb.capacity());//default 16

		sb.append("Hello");
		System.out.println(sb.capacity());//now 16

		sb.append("java is my favourite language");
		System.out.println(sb.capacity());//now (16*2)+2=34 i.e (oldcapacity*2)+2

	}
  }

    public void ensureCapacity(int minimumCapacity): It is used to ensure that the minimumcapacity is less than or equal to the current capacity of the invoking StringBuffer object. If not the current capacity increases to (currentcapacity*2) +2 and goes on increasing until it becomes greater than or equal to the minimumcapacity. 
      //Example of ensureCapacity() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuffer sb= new StringBuffer();
		System.out.println(sb.capacity());//default 16

		sb.ensureCapacity(10);//now no change
		System.out.println(sb.capacity());//now 16

		sb.ensureCapacity(30);//now (16*2)+2
		System.out.println(sb.capacity());//now 34

	}
  }
