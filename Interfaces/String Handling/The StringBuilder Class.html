The StringBuilder Class
  • The java.lang.StringBuilder class should be used when you have to make a lot of modifications to strings of characters because StringBuilder objects are mutable in nature.
  • The StringBuilder class is same as String except it is mutable i.e. its value can be changed.
  • A common use for StringBuilder is file I/O when large, ever-changing streams of input are being handled by the program. In these cases, large blocks of characters are handled as units, and StringBuilder objects are the ideal way to handle a block of data, pass it on, and then reuse the same memory to handle the next block of data.
  • The java.lang.StringBuilder class is final in nature.
  • The StringBuilder class was added in Java 5. It has exactly the same API as the StringBuffer class, except StringBuffer is not thread safe. In other words, its methods are not synchronized.
  • Sun recommends that you use StringBuilder instead of StringBuffer whenever possible because StringBuilder will run faster.
  • Some commonly used constructors of StringBuilder class are:
    1. StringBuilder() : creates an empty StringBuilder object with the initial capacity of 16.
    2. StringBuilder(String str) : creates a StringBuilder object with the specified string.
    3. StringBuilder(int capacity) : creates an empty StringBuilder object with the specified capacity as length.
  • Some commonly used methods of StringBuilder class are:

    public StringBuilder append(String s): It is used to append the argument passed with the invoking string. The append() method is overloaded like append(char), append(boolean), append(int), append(float), append(double) etc. 
      //Example of append() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuilder sb = new StringBuilder("set ");
		sb.append("point");
		System.out.println(sb); // output is "set point" 

		StringBuilder sb2 = new StringBuilder("pi = ");
		sb2.append(3.14159f);
		System.out.println(sb2); // output is "pi = 3.14159"
	}
  }

      //Example of insert() method

    public int capacity(): It is used to return the current capacity of the invoking StringBuilder object. The default capacity of a StringBuilder object is 16. If the length of the object becomes more than the current capacity of the object due to modification then the capacity increases to (currentcapacity*2) +2. 
      //Example of capacity() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuilder sb= new StringBuilder();
		System.out.println(sb.capacity());//default 16

		sb.append("Hello");
		System.out.println(sb.capacity());//now 16

		sb.append("java is my favourite language");
		System.out.println(sb.capacity());//now (16*2)+2=34 i.e (oldcapacity*2)+2

	}
  }

    public void ensureCapacity(int minimumCapacity): It is used to ensure that the minimumcapacity is less than or equal to the current capacity of the invoking StringBuilder object. If not the current capacity increases to (currentcapacity*2) +2 and goes on increasing until it becomes greater than or equal to the minimumcapacity. 
      //Example of ensureCapacity() method
												
  class Simple{
  	public static void main(String args[]){

		StringBuilder sb= new StringBuilder();
		System.out.println(sb.capacity());//default 16

		sb.ensureCapacity(10);//now no change
		System.out.println(sb.capacity());//now 16

		sb.ensureCapacity(30);//now (16*2)+2
		System.out.println(sb.capacity());//now 34

	}
  }
